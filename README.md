# memorg

Внешняя память для Claude Code: система, в которой длинная работа с агентом
переживает автокомпакты, новые сессии и смену машин. Модульная: репа — общий
каталог (marketplace), каждый слой системы — отдельный плагин, ставится
независимо.

## Слои

| Плагин | Что делает | Статус |
|---|---|---|
| **discussions** | Длинные обсуждения: живая карта решений (WIP) + накапливаемый финал (design), автоякорь в CLAUDE.md | ✅ доступен |
| memory | Стандарт файлов памяти (frontmatter + индексы) | планируется |
| knowledge-bank | Структурированная база знаний проекта | планируется |

## Установка

Вариант для человека — в Claude Code выполнить:

```
/plugin marketplace add SkyTger/memorg
/plugin install discussions@memorg
```

Вариант для агента — добавить в `~/.claude/settings.json`:

```json
{
  "extraKnownMarketplaces": {
    "memorg": {
      "source": { "source": "github", "repo": "SkyTger/memorg" },
      "autoUpdate": true
    }
  },
  "enabledPlugins": { "discussions@memorg": true }
}
```

## Плагин discussions — как пользоваться

1. Открой свою рабочую папку в Claude Code (всегда одну и ту же для
   одной темы).
2. Новая большая тема — скажи `/discussion-new` (агент спросит имя).
3. Дальше просто разговаривай. Агент ведёт карту решений сам; после
   сбоя, сжатия контекста или новой сессии он вернётся в тему по карте.
4. Тема доведена до финала — `/discussion-close`: сверка полноты,
   финальный документ, архив.

Файлы обсуждений — обычный markdown в папке `discussions/` твоего проекта:
читаются человеком, едут с git, не привязаны к инструменту.

## Ограничения

- Работает в Claude Code (CLI, desktop, IDE-расширения). Другие агентные
  системы (Codex, Gemini CLI и т.п.) плагины Claude Code не читают;
  перенос концепции туда — отдельная адаптация инструкций под каждый
  инструмент (в планах при расширении).
- Облачная web-версия Claude Code плагины не поддерживает.
