# memorg

> 🇷🇺 Русский · [🇬🇧 English](README.en.md)

**Внешняя память для Claude Code.** Длинная работа с агентом переживает
сжатие контекста (автокомпакты), новые сессии и смену машин — потому что
живёт не в контексте разговора, а в обычных markdown-файлах рядом с твоим
проектом.

Репа устроена как каталог плагинов (marketplace). Сейчас в ней один
плагин — **discussions**; планируется расширение.

## Проблема

Любой долгий диалог с агентом упирается в предел контекста: сессия
«сжимается» или начинается заново — и агент забывает половину принятых
решений, ходит по кругу, переспрашивает. Особенно больно в длинных
проработках: дизайн системы, исследование, продумывание контент-плана,
юридического документа — чего угодно на десятки раундов.

## Решение: плагин discussions

Каждая большая тема получает свою папку с двумя файлами:

- **WIP.md** — живая карта: что решено, что открыто, что уже сделано.
  Агент обновляет её сам по ходу разговора.
- **design.md** — финал, который дозревает к закрытию темы.

Плюс автоматика, ради которой это плагин, а не просто привычка:

- `/discussion-new` сам создаёт структуру **и прописывает якорь в
  CLAUDE.md** проекта — карта обсуждения гарантированно оказывается в
  контексте каждой новой сессии, без надежды на память агента.
- Правила сверки: агент не объявляет вопрос закрытым, не перепроверив
  карту; `/discussion-close` закрывает тему только после полной ревизии
  (все ли решения записаны, всё ли открытое закрыто осознанно).

После сжатия контекста или в новой сессии ничего восстанавливать не надо:
агент возвращается в тему по карте и продолжает с того же места.

## Установка

**Claude Desktop (приложение):** File → Settings → Plugins → кнопка **+**
→ указать репозиторий `SkyTger/memorg` → установить **discussions**.
Перезапустить приложение. Команды плагина появятся во всех вкладках,
включая Code.

**Claude Code (терминал или IDE-расширение):**

```
/plugin marketplace add SkyTger/memorg
/plugin install discussions@memorg
```

Либо, если настройку делает агент — добавить в `~/.claude/settings.json`:

```json
{
  "extraKnownMarketplaces": {
    "memorg": {
      "source": { "source": "github", "repo": "SkyTger/memorg" },
      "autoUpdate": true
    }
  },
  "enabledPlugins": { "discussions@memorg": true }
}
```

После установки перезапусти сессию.

## Как пользоваться

1. Открой рабочую папку проекта (для одной темы — всегда одну и ту же):
   в терминале — `cd` в папку и `claude`, в приложении — вкладка **Code**
   → выбор папки.
2. Новая большая тема — `/discussion-new` (можно сразу с темой:
   `/discussion-new правила обезличивания`). Агент заведёт структуру сам
   — включая папку `discussions/`, если её ещё нет, — а если тема уже
   обсуждалась в чате, сразу заполнит карту из разговора.
3. Дальше просто разговаривай. Карту ведёт агент.
4. Тема дозрела — `/discussion-close`: сверка полноты, финальный
   документ, архив.

Файлы обсуждений лежат в `discussions/` внутри твоего проекта:

```
мой-проект/
├── CLAUDE.md                  ← якорь на активные обсуждения (ведётся само)
└── discussions/
    ├── INDEX.md               ← список тем
    └── моя-тема/
        ├── WIP.md             ← живая карта
        ├── design.md          ← финал (после закрытия)
        └── notes/             ← материалы, сырьё
```

Это обычный markdown: читается человеком, версионируется git'ом вместе с
проектом, не привязан к инструменту.

## Где работает

| Среда | Статус |
|---|---|
| Claude Code CLI (терминал) | полностью |
| Claude Desktop, вкладка Code | полностью — это тот же локальный Claude Code |
| IDE-расширения (VS Code, JetBrains) | полностью |
| Cowork (десктоп) | с подключённой рабочей папкой; обсуждения живут в ней |

В Cowork между сессиями сохраняется только подключённая папка — перед
`/discussion-new` подключи папку, в которой должна жить тема.

## Ограничения

- Другие агентные системы (Codex, Gemini CLI и др.) плагины Claude Code
  не читают — перенос концепции туда планируется отдельными адаптациями.
  Сами файлы обсуждений при этом переносимы уже сейчас — это просто
  markdown.
- Облачная web-версия Claude Code плагины не поддерживает.
