# /discussion-handoff — Экспорт обсуждения

> Создаёт самодостаточный handoff-файл: выжимку обсуждения, по которой
> человек или агент, не видевший беседы, может включиться в тему с нуля.
> Перед экспортом WIP сверяется с текущим состоянием беседы (антидрифт).

**Когда команда нужна, а когда нет.** Если папка проекта синхронизируется
через git и на другой стороне делают `git pull` — переносом занимается
сам git, handoff не нужен. Команда — для случаев, когда общего git нет:
проект без репозитория, перенос через мессенджер/облако/флешку, передача
темы коллеге без доступа к репе. Если пользователь просит handoff там,
где git и так всё переносит, — скажи об этом одной строкой и выполняй.

## Аргументы

Вход — тема в свободной форме или имя папки обсуждения. Сопоставь её с
активными обсуждениями из `INDEX.md` и определи `<topic-slug>` сам. Если
аргумент не передан или совпадение неоднозначно — покажи список активных
обсуждений человеческим языком (описание + имя папки) и дай выбрать.
Технические термины («slug», «kebab-case») в диалоге не употреблять.

## Алгоритм

### 1. Найди обсуждение

Та же логика, что в `/discussion-close`:

```bash
PROJ_ROOT=$(git rev-parse --show-toplevel 2>/dev/null || pwd)
DISC=""
for CAND in "$PROJ_ROOT/.obsidian-docs/discussions" "$PROJ_ROOT/discussions"; do
  [ -d "$CAND/<topic-slug>" ] && DISC="$CAND/<topic-slug>" && break
done
test -f "$DISC/WIP.md" || { echo "WIP.md отсутствует"; exit; }
```

Если в текущем месте не найдено — спроси пользователя, в какой папке
ведётся обсуждение (место определяется темой, не сессией), и продолжай
оттуда.

### 2. Антидрифт — ОБЯЗАТЕЛЬНЫЙ шаг

**Нельзя пропускать.** Перед экспортом:

1. Прочитай `WIP.md` целиком.
2. Сверь с текущим состоянием беседы в этой сессии:
   - Есть ли решения, принятые в разговоре, но не отражённые в WIP?
   - Есть ли открытые вопросы, которые были закрыты, но WIP не обновлён?
   - Есть ли новый контекст или аргументы, не записанные в WIP?
3. Если расхождения найдены — **сначала обнови WIP.md**, затем продолжай.
4. Если расхождений нет — скажи: «WIP актуален, расхождений не найдено.»

> Не спрашивай «обновить WIP?» — обновляй сразу. Спрашивай только если
> не уверен в интерпретации решения.

### 3. Прочитай все артефакты обсуждения

Все файлы в папке обсуждения: WIP.md, design.md (если есть), notes/ и
прочее — handoff собирается по полному материалу, не только по WIP.

### 4. Сгенерируй handoff-файл

Файл кладётся **в саму папку обсуждения** — он виден человеку, едет
вместе с проектом и его удобно переслать любым каналом.
`$DISC/handoff.md` (перезаписывай, если существует — handoff всегда
отражает последнее состояние):

```markdown
---
name: <Topic> handoff
description: <одно предложение — что обсуждаем>
type: project
exported: <YYYY-MM-DD>
status: active
---
# <Topic> — Handoff

> Самодостаточная выжимка обсуждения для продолжения на другой машине
> или другим человеком. Импорт: положить файл в рабочую папку и вызвать
> `/discussion-new <тема>` — агент развернёт обсуждение из этого файла.

## Контекст

<Зачем обсуждение возникло. Предыстория. Что важно знать, чтобы
включиться с нуля. 2-4 абзаца — не экономь на контексте: это
единственный источник правды на принимающей стороне.>

## Ключевые решения

<Пронумерованный список решений с краткой аргументацией по каждому.
Формат: решение → почему выбрали именно так (1-2 предложения).>

## Открытые вопросы

<Пронумерованный список вопросов, которые ещё не закрыты. Для каждого —
текущие варианты и к чему склоняемся (если есть).>

## Ход рассуждений

<Ключевые аргументы, отвергнутые варианты и почему. Это то, что
теряется при пересказе «только решений». 3-5 самых важных поворотов
обсуждения.>

## Применённые изменения

<Что физически сделано в файлах/конфигах. Пути, описание изменений.
Если ничего — так и написать.>
```

### 5. Отчёт пользователю

```
Handoff создан: <путь к $DISC>/handoff.md

- Антидрифт: <WIP актуален | WIP обновлён — добавлено N решений/вопросов>
- Собрано из: <перечисление файлов обсуждения>

Дальше: перешли файл (или папку обсуждения целиком) любым каналом.
На принимающей стороне: положить его в рабочую папку и вызвать
/discussion-new <тема> — обсуждение развернётся из handoff.
```

## Правила

- **Не печатай содержимое handoff в чат.** Только статус и путь.
- Handoff — это НЕ копия WIP. Это переработанное изложение для человека
  (или агента), который не видел беседу. Пиши так, чтобы было понятно
  без дополнительного контекста.
- Одно обсуждение = один handoff-файл. Повторный экспорт перезаписывает
  предыдущий.

## Если пользователь спросит, что это за команда

Первый ответ — каноническим текстом (дословно или очень близко):

> Это упаковка обсуждения в один файл для переноса — на другой компьютер
> или другому человеку. Я сверяю живую карту с разговором, собираю
> выжимку (контекст, решения, открытые вопросы) и кладу её в папку темы.
> Если папка проекта и так синхронизируется через git — команда не
> нужна: перенос делает git.

Дальше — своими словами по тому же правилу: без технических терминов
(WIP, frontmatter, slug, якорь, CLAUDE.md). Технические подробности —
только если спросят прямо.
